Selected Acheivements

Experienced and proven as an expert in Single Page Applications (SPA), UX, UI, ID, full-stack architecture, and development. CEO and founder of two startups.

Works to relentlessly reduce cycle times, fail faster, and deliver a better product sooner. Here are a few objectives to meet this goal:

  • Focus on language mastery instead of silver bullet frameworks and libraries with complex and buggy Domain Specific Languages [DSLs]. Use fewer, high quality, focused, and mature tools instead.
  • Strive for a "one-touch" build system. Build times for a JS project should not exceed 5 minute. Ten seconds is even better.
  • Architect and code for TDD. Don't use TDD too early in a project because there's little value in testing prototype code. But when you get do use TDD, a little forethought can cut cycle times by days.
  • Strive for automatic unit tests that provide good coverage and run fast. Don't test system logic with incredibly expensive UI tests (e.g. Selinium). Use headless testing for that.
  • Use commit hooks to block sub-par code. The commit hook should, at the minimum, run regression tests and lint tests on all production code.
  • Use a SCMS that encourages branching.
  • Favor a smaller, focused, full-time, colocated team over larger, part-time, dispersed team. If the team can't share the same facilities, the closer the time zones the better.
  • Embrace meaningful and clear code standards so that developers can collaborate.
  • Employ strategic pair-programming and peer code review.
  • Be pragmatic, not religious. Get stuff done by leaving comments for future improvements instead of getting stuck on optimizing too early.
